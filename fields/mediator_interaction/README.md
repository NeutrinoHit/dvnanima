# Scalar Electrodynamics With `A_0`

Этот файл фиксирует постановку задачи для полной переписи проекта `mediator_interaction`.
На этом этапе это именно спецификация модели и численного метода. Код под неё будет переписан отдельно после согласования.

## Идея

Нужно показать, как локализованные заряженные волновые пакеты взаимодействуют через электростатический посредник `A_0`.

Вместо искусственной модели двух скалярных полей `phi-chi` берём более физически прозрачную систему:

- материальное поле: комплексное скалярное поле `\varphi(t,x,y)`;
- посредник: электромагнитный потенциал `A_0(t,x,y)`;
- пространственные компоненты `A_i` на первом этапе не включаем.

То есть это не полная QED, а электростатический сектор scalar electrodynamics в `(2+1)` измерениях.

## Лагранжиан

Берём стандартный лагранжиан комплексного скалярного поля с минимальным калибровочным взаимодействием:

```math
\mathcal L
=
(D_\mu \varphi)^* D^\mu \varphi
- m^2 |\varphi|^2
- \frac14 F_{\mu\nu}F^{\mu\nu},
\qquad
D_\mu = \partial_\mu + i e A_\mu.
```

На первом этапе делаем электростатическое приближение:

```math
A_i = 0,
\qquad
A_0 = A_0(t,x,y).
```

Тогда

```math
D_0 = \partial_t + i e A_0,
\qquad
D_i = \partial_i.
```

## Уравнения движения

Поле материи:

```math
D_\mu D^\mu \varphi + m^2 \varphi = 0.
```

В нашем секторе это даёт

```math
(\partial_t + i e A_0)^2 \varphi - \nabla^2 \varphi + m^2 \varphi = 0.
```

Уравнение Гаусса:

```math
\partial_\mu F^{\mu 0} = j^0,
```

где зарядовая плотность

```math
j^0 = - i e \left(\varphi^* D_0 \varphi - (D_0 \varphi)^* \varphi \right).
```

Так как `A_i = 0`, получаем эллиптическое уравнение на `A_0`:

```math
-\nabla^2 A_0 = j^0.
```

Это уже достаточная система, но для численного решения удобнее перейти к переменной

```math
\pi = D_0 \varphi.
```

Тогда система принимает вид

```math
\partial_t \varphi = \pi - i e A_0 \varphi,
```

```math
\partial_t \pi = \nabla^2 \varphi - m^2 \varphi - i e A_0 \pi,
```

а уравнение Гаусса упрощается до

```math
-\nabla^2 A_0 = - i e \left(\varphi^* \pi - \pi^* \varphi \right).
```

Это ключевая форма для численной реализации.

## Начальные условия

Для первого рабочего варианта разумно брать два локализованных узкополосных пакета:

```math
\varphi(0,\mathbf r) = \varphi_1(\mathbf r) + \varphi_2(\mathbf r),
```

где каждый пакет имеет вид

```math
\varphi_a(\mathbf r)
=
\mathcal A_a
\exp\!\left(
-\frac{|\mathbf r - \mathbf r_a|^2}{2\sigma^2}
\right)
e^{i \mathbf k_a \cdot (\mathbf r - \mathbf r_a)}.
```

Нужно отдельно выбрать зарядовую структуру пакетов.

### Важный численный момент

Если решать уравнение Гаусса через FFT на периодической области, то нулевая мода `k=0` делает задачу чувствительной к полному заряду.
Поэтому для первого устойчивого варианта лучше брать **суммарно нейтральную конфигурацию**.

Самый чистый стартовый выбор:

- один положительно заряженный пакет;
- один отрицательно заряженный пакет;
- полный заряд области близок к нулю.

Тогда Poisson solve для `A_0` через FFT становится естественным.

Если позже понадобится моделировать рассеяние одинаковых зарядов, есть два варианта:

1. вводить нейтрализующий фон;
2. уходить от чисто периодического Poisson solve к граничной задаче на конечной области.

Для первой версии проекта лучше выбрать нейтральную пару.

## Численный метод

### 1. Пространственная дискретизация

Работаем на прямоугольной сетке по `x,y`.

Для лапласиана используем псевдоспектральный метод:

```math
\widehat{\nabla^2 f} = -(k_x^2 + k_y^2)\hat f.
```

Это даёт хорошую точность на гладких пакетах.

### 2. Эволюция `\varphi, \pi`

На первом этапе достаточно явной схемы второго порядка по времени:

```math
\partial_t \varphi = \pi - i e A_0 \varphi,
\qquad
\partial_t \pi = \nabla^2 \varphi - m^2 \varphi - i e A_0 \pi.
```

Практически это можно реализовать как:

- либо `RK4` для пары `(\varphi,\pi)`;
- либо leapfrog / Stormer-Verlet-подобную схему;
- при каждом подшаге нужно пересчитывать `A_0` из уравнения Гаусса.

Для первой версии я бы выбрал:

- `RK4` по времени для `(\varphi,\pi)`;
- спектральный solve для `A_0` на каждом substep.

Причина простая: код будет чуть тяжелее, но понятнее и стабильнее, чем пытаться сразу аккуратно собирать симплектическую схему с калибровочным ограничением.

### 3. Решение уравнения Гаусса

В каждый момент времени:

```math
\rho = - i e \left(\varphi^* \pi - \pi^* \varphi \right),
```

далее решаем

```math
-\nabla^2 A_0 = \rho.
```

В `k`-пространстве:

```math
\hat A_0(\mathbf k) = \frac{\hat\rho(\mathbf k)}{k^2},
\qquad k \neq 0.
```

Нулевую моду надо обработать отдельно.
Для нейтральной конфигурации естественно положить

```math
\hat A_0(\mathbf 0) = 0.
```

## Граничные условия

На первом этапе:

- периодическая область для spectral method;
- мягкое damping-окно у краёв только если понадобится подавить обратное входение пакетов в кадр.

Но важно: damping не должен участвовать в основной физике столкновения.
Зона взаимодействия должна находиться далеко от края.

## Что будет считаться успехом первой версии

Первая версия считается удачной, если:

1. два пакета устойчиво эволюционируют без артефактного blow-up;
2. `A_0` отчётливо виден как поле, создаваемое зарядом;
3. при ненулевом impact parameter видно отклонение траекторий;
4. результат сохраняется при разумочном refinement по `dt`, `nx`, `ny`.

   - `analyze_scalar_ed.py`

5. Только после этого подключать визуализацию:
   - `|\varphi|^2`,
   - `A_0`,
   - при необходимости траектории центров.
